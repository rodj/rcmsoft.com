Getting Started with Claude Code

By Robert Martin — RCMSoft — [PUBLISH DATE]

If you've been curious about AI coding assistants but found tools like GitHub Copilot a bit shallow — good for autocomplete but not much more — Claude Code is worth a closer look. It's a command-line tool from Anthropic that gives you a genuine AI collaborator in your terminal: one that can read your entire codebase, reason about it, write multi-file changes, run tests, and work through problems iteratively.

This article is a practical introduction. I'll explain what Claude Code actually is, how it differs from other AI coding tools, how to get it running, and how to use it effectively from day one.

What is Claude Code?

Claude Code is a CLI (command-line interface) tool that wraps Anthropic's Claude AI model with a set of capabilities specifically designed for software development. Unlike a chat interface where you paste code snippets back and forth, Claude Code runs in your terminal and has direct access to your project files. It can:

  • Read any file in your project without you having to paste it
  • Make edits to multiple files in a single operation
  • Run shell commands, tests, and build scripts
  • Search your codebase by file pattern or content
  • Use external tools via MCP (Model Context Protocol) servers

The result is something that feels much closer to pair programming with a knowledgeable colleague than to a glorified autocomplete engine.

How it differs from GitHub Copilot

Copilot lives inside your editor and makes inline suggestions as you type. It's excellent for that use case — filling in boilerplate, completing function signatures, suggesting common patterns. But it works at the scope of what's visible in your current file and doesn't reason across your whole project.

Claude Code works at a higher level of abstraction. You describe what you want to accomplish and it figures out what files to touch, what changes to make, and how they fit together. You're directing the work rather than accepting or rejecting individual suggestions. The two tools complement each other well — use Copilot for line-level editing and Claude Code for feature-level changes.

Installation

Claude Code requires Node.js 18 or higher. Install it with npm:

npm install -g @anthropic-ai/claude-code

After installation, run claude from any directory to start an interactive session. The first time you run it, you'll be prompted to authenticate with your Anthropic account. You'll need an Anthropic API key, which requires a paid account (as of 2026, there's no free tier for Claude Code).

Pro Tip Run claude from the root of your project, not a subfolder. Claude Code will automatically explore your directory tree from wherever you launch it, and starting at the root gives it the best context about your project structure.

Your first session

Once you've launched Claude Code in a project directory, you can just describe what you want in plain English. Some examples of effective prompts to start with:

  • "Explain how this project is structured and what the main components do." — Good for getting oriented before making changes.
  • "Find all places where we connect to the database and tell me what connection string format we're using." — Good for understanding patterns before you change them.
  • "Add a new API endpoint that returns a list of customers filtered by city. Follow the same pattern as the existing endpoints." — Good for feature additions that should follow established conventions.

The CLAUDE.md file

One of the most useful features for ongoing projects is the CLAUDE.md file. If you place a file with this name in your project root (or any subdirectory), Claude Code will automatically read it at the start of every session. Use it to document:

  • How to build and run the project
  • Architectural conventions and decisions you want followed
  • Things Claude should never do in this codebase (e.g., "never commit directly to main")
  • Key file locations and what they contain

A good CLAUDE.md is the difference between Claude Code needing to rediscover your project's conventions on every session versus picking up exactly where you left off.

Practical tips

Be specific about constraints. Instead of "add error handling," say "add error handling that logs to our existing logger at /src/utils/logger.ts and returns a 400 status for validation errors, 500 for everything else." The more context you give, the less back-and-forth you'll need.

Ask for a plan before execution. For larger changes, ask Claude Code to describe what it plans to do before it does it. This catches misunderstandings early: "Before making any changes, tell me which files you'll touch and what you'll change in each."

Use it for reading, not just writing. Claude Code's ability to search and summarize your codebase is genuinely valuable even when you're not making changes. "How does authentication work in this app?" can save 20 minutes of manual file reading.

Review diffs before moving on. Claude Code shows you what it changed. Actually read those changes. It's almost always correct, but "almost always" isn't the same as "always," and a five-second review of a diff is much cheaper than debugging an incorrect assumption later.

When to use it, when not to

Claude Code is excellent for: implementing features you've already thought through, refactoring toward a pattern you can describe, writing tests for existing code, and understanding unfamiliar codebases. It's less suited to open-ended architectural exploration where the requirements themselves are unclear — that kind of thinking is still best done between humans before involving an AI tool.

Getting more out of it

Once you're comfortable with the basics, look into MCP (Model Context Protocol) servers — Claude Code's plugin system that lets it connect to external tools like databases, APIs, and web services. In a future article I'll walk through building a custom MCP server to automate Microsoft Word document creation, which is a great example of what's possible when you extend Claude Code's reach beyond your codebase.
